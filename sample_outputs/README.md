# sample_outputs/ — Pre-Generated Scan Outputs

This folder contains pre-generated outputs from Aegis-API scans run against
deliberately vulnerable lab APIs. These files exist so that recruiters,
interviewers, and GitHub visitors can see exactly what the tool produces
**without needing to run it themselves.**

All target data in these outputs is **fake and anonymized**. No real URLs,
real tokens, or real personal data appear anywhere in these files.

---

## What Each File Contains

### `sample_VAPT_Report.pdf`

A complete professional Vulnerability Assessment and Penetration Testing (VAPT)
report generated by Aegis-API's unified PDF report engine (`utils/reporting.py`).

**Demonstrates:**
- REST findings from crAPI: 2 IDOR/BOLA, 1 Mass Assignment, 1 JWT weakness, 1 Rate Limit
- SOAP findings from WebGoat: 1 XXE Injection, 1 WSDL Enumeration (information disclosure)
- GraphQL findings from DVGA: 1 Introspection Abuse, 1 Query Depth Attack, 1 Batching Abuse

**Report sections shown:**
1. Executive Summary with overall risk rating
2. Scope and Methodology with protocol detection notes
3. OWASP API Top 10 coverage table (REST / SOAP / GraphQL columns)
4. Detailed findings with CVSS scores, attack reproduction steps, request/response evidence
5. Remediation Summary ranked by severity
6. Appendix with raw logs

**Format:** PDF, generated via `reportlab` and `fpdf2`

---

### `sample_audit_log.csv`

The CSV audit log written by `utils/logger.py` during a full multi-protocol scan session.

**Demonstrates:**
- REST GET and PUT request entries (IDOR and Mass Assignment tests)
- SOAP POST entries with SOAPAction header values
- GraphQL POST entries with introspection and depth query payloads
- Timestamps per request
- HTTP status codes and response size columns
- Finding severity column where a vulnerability was confirmed

**Columns:**
```
timestamp, protocol, method, url, status_code, response_size_bytes,
test_type, finding, severity, owasp_id
```

**Useful for:** showing that Aegis-API logs all traffic systematically —
which is what any real penetration testing engagement requires for audit trail.

## How to Regenerate These Outputs

To generate fresh outputs from scratch against the live lab environments:

### Step 1 — Start all three lab environments

```bash
# Terminal 1 — REST lab
docker-compose -f crapi-docker-compose.yml up -d

# Terminal 2 — SOAP lab
docker run -p 8080:8080 -p 9090:9090 -e WEBGOAT_HOST=0.0.0.0 webgoat/goat-and-wolf

# Terminal 3 — GraphQL lab
docker run -t -p 5013:5013 -e WEB_HOST=0.0.0.0 dolevf/dvga
```

### Step 2 — Run Aegis-API against each lab

```bash
# REST scan (crAPI)
python main.py --url http://localhost:8888/api --protocol rest --token YOUR_JWT

# SOAP scan (WebGoat)
python main.py --url http://localhost:8080/WebGoat/services --protocol soap

# GraphQL scan (DVGA)
python main.py --url http://localhost:5013/graphql --protocol graphql
```

### Step 3 — Collect outputs

After each scan completes, outputs are written to `reports/` and `audit_log.csv`.
Copy the final PDF and CSV into `sample_outputs/` and rename them to the
`sample_` prefix to keep them as reference artifacts.

| Lab | Protocol | Docker Hub | Purpose |
|-----|----------|------------|---------|
| [crAPI](https://github.com/OWASP/crAPI) | REST | `docker pull crapi/crapi` | IDOR, Mass Assignment, JWT, Rate Limit |

All three are official or widely-recognized deliberately vulnerable applications.
They are safe to run locally on Docker for learning and testing purposes.
Never expose them on a public network or production machine.
